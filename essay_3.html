Editors (Vim)
Modes:

  • Insert (i)
  • Replace (R)
  • Visual mode (v)
  • Visual Line mode (V)
  • Visual Block mode (Ctrl + v)
  • Insert (i)
  • Command line (:)

Command line

  • :q quit (close window)
  • :w save (“write”)
  • :wq save and quit
  • :e {name of file} open file for editing
  • :ls show open buffers
  • :help {topic} open help
  • :help :w opens help for the :w command
  • :help w opens help for the w movement

Movement

  • (left, down, up, right) (hjkl)
  • Words: w (next word), b (beginning of word), e (end of word)
  • Lines: 0 (beginning of line), ^ (first non-blank character), $ (end of line)
  • Screen: H (top of screen), M (middle of screen), L (bottom of screen)
  • Scroll: Ctrl-u (up), Ctrl-d (down)
  • File: gg (beginning of file), G (end of file)
  • Line numbers: {number}G (line {number})
  • Find (f/F)
  • To (t/T)

Edits

  • insert below (o) above (O)
  • Delete d{motion} dw is delete word, d$ is delete to end of line, d0 is delete to beginning of line
  • Change c{motion} cw is change word
  • Delete character (x)
  • Substitute character (s)
  • Undo(Ctrl + u)
  • Copy (y)
  • Paster (p)
  • /search
  • ci{charcter}

Counts

  • 3w move 3 words forward
  • 5j move 5 lines down
  • 7dw delete 7 words

Test Driven Development
Test Driven Development or TDD is a idealy workflow when you have to make tests between each stage of you lifecycle development, there are 5 stages and this worflow or loop for developing looks like this:

  • Code
    • Test
  • Integrate
    • Test
  • Deploy
    • Test
  • Release
    • Test
  • Steer
    • Test
  • Repeat

Some recomendations for doing this tests or better code are:

  • DRY (Do not Repeat Yourself)
  • Make modular (every module only does one thing)
  • Unit Testing
  • Refactor
  • When coding think about what you want to do
  • When coding think about how to test your code
  • Run test and then fix if there were errors or bugs

Java Build Tools for Dependency Management:
I learned about differents tools for managing dependencies in Java, but Maven was the most remarkable; Here there is a guide for Maven and Gradel installation, these are the basic commands:

  • mvn compile
  • mvn test
  • mvn package
  • mvn help:describe -Dplugin=org.apache.tomcat.maven:tomcat7-maven-plugin
  • mvn deploy
  • mvn dependency:tree -Dverbose

This is the build cycle of maven

  • validate - validate the project is correct and all necessary information is available
  • compile - compile the source code of the project
  • test - test the compiled source code using a suitable unit testing framework. These tests should not require the code packaged or deployed
  • package - take the compiled code and package it in its distributable format, such as a JAR.
  • verify - run any checks on results of integration tests to ensure quality criteria are met
  • install - install the package into the local repository, for use as a dependency in other projects locally
  • deploy- done in the build environment, copies the final package to the remote repository for sharing with other developers and projects.
